$\Huge{Задание}$

Задача: реализовать алгоритм перемножения матриц

Язык: C++ или Python

Входные данные: 2 матрицы размером от 100х100 до 2000х2000 каждая.
Выходные данные: проверка корректности перемножения + время вычисления
Реализация должна содержать 2 функции перемножения матриц: на CPU и на GPU с
применением CUDA.

Отчет о проделанной лабораторной работе - это git-репозиторий с исходным кодом
реализации + описание проделанной работы там же в readme.
Необходимо описать реализацию, объяснив, что https://github.com/Alexeyzse4r/HPC_labs/tree/master/lab1конкретно было распараллелено и
почему.

Провести эксперименты: перемножить матрицы разных размеров, посчитать
ускорение. Результаты привести в виде таблицы/графика.


$\Huge{Работа}$

Лабораторная работа реализована на C++.

Умножение двух матриц NхN на CPU(cpu_matmul) выполняется стандартно с помощью трех вложенных циклов. Каждый элемент итоговой матрицы вычисляется как скалярное произведение строки 𝑖 матрицы 𝐴 и столбца 𝑗 матрицы 𝐵

Умножение двух матриц NхN на GPU(gpu_matmul) выполняется с испольльзованием CUDA. Вычисления распределяются между потоками: каждый поток считает один элемент итоговой матрицы
(
индексация:
    int i = blockIdx.y * blockDim.y + threadIdx.y;
    int j = blockIdx.x * blockDim.x + threadIdx.x;
).

Для получения достоверных результатов каждый тест(размерность матриц) выполняется 30 раз(NUM_RUNS) и время выполнения усредняется.

По результатам видно что GPU при увеличении размерности задачи значительно превосходит по скорости CPU. Время выполнения на CPU растет с 0.0003с при N=100 до 14.51с при N=2000, в то время как на GPU(даже вместе с пересылками) с  0.0004с до 0.29с.
